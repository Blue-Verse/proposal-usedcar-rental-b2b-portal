프로젝트 소개

관리자 백오피스와 에이전트 포털을 단일 아키텍처로 통합 구축. 테넌트 격리·민감정보 마스킹 기반 금융권 수준 보안과 목록 2초·상세 1.5초·동시 50명을 보장하는 엔터프라이즈급 성능을 안정적으로 구현합니다.

핵심 요구사항 분석

  • 관리자·에이전트 이원화 인증 체계 및 에이전트별 테넌트 격리 적용
  • 민감정보 마스킹 및 권한별 접근 제한(RBAC)
  • 차량 전주기 관리: 다중 사진 업로드·이력 관리, 계약 상태 전이 처리, 계약서·서류 파일 업로드
  • 고객 정보 및 상담 이력 관리, 에이전트 계정·권한 설정, 차량·계약 현황 대시보드
  • 에이전트 포털: 신용/저신용 탭 분리 카탈로그, 다중 필터·정렬, URL 라우팅 보안 상세 조회, 이미지 리사이즈 스트리밍
  • 렌탈 조건별 견적 확인, 계약 요청 전송, 본인 계약 목록 조회 및 상태 시각화
  • 카카오 알림톡 템플릿 연동·발송, 레거시 시스템 전송 API, 차량 카운트 API Redis 캐싱
  • 성능: 목록 2초·상세 1.5초 이내, 동시 50명 안정성 / TypeScript strict·NestJS 모듈 단위 아키텍처

제안 포인트

① 테넌트 격리 + 민감정보 마스킹을 설계 단계부터 내재화 — 다수 에이전트가 동시 접속하는 환경에서 데이터 누수 없는 완전한 격리가 본 프로젝트의 최우선 리스크입니다. 멀티테넌트 아키텍처와 5계층 보안(DLP 포함)을 실제로 운영한 수행 경험을 바탕으로, NestJS Guard/Interceptor 계층에서 테넌트 스코프와 필드 단위 마스킹을 일관되게 강제하는 구조로 착수 4주 내 보안 모듈을 우선 구축합니다.

② 이원화 인증과 권한별 접근 제한을 RBAC로 정형화 — 관리자/에이전트 인증 도메인을 분리하고 JWT 기반 권한 모델을 역할·리소스 단위로 정의하여, "본인 계약만 조회", "신용/저신용 탭 접근 제한" 등 화면·API 양쪽에서 동일 정책이 적용되도록 설계합니다.

③ 성능 목표(2초/1.5초/동시 50명)를 정량 검증으로 보장 — 차량 카운트 API Redis 캐싱, 목록 커서 페이지네이션, 이미지 리사이즈 스트리밍(on-the-fly resize + CDN 캐시)을 적용하고, 통합 테스트 단계에서 동시 50명 부하 시나리오로 목표 응답시간을 수치로 입증합니다.

④ 외부 연동을 모듈 경계로 분리해 레거시 리스크 차단 — 카카오 알림톡과 레거시 전송 API를 어댑터 모듈로 격리하고 재시도·실패 로깅을 표준화하여, 외부 장애가 핵심 플랫폼으로 전파되지 않도록 합니다.

⑤ 2개 플랫폼 동시 구축 역량 — 공고에 명시된 자매 프로젝트까지 고려한 통합 데이터 모델·인증 기반 설계로, 향후 확장 시에도 일관성을 유지합니다.

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기능 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

내부 업무 효율화를 위한 고도화된 관리자 백오피스와, 외부 파트너 에이전트가 차량을 탐색하고 계약을 요청하는 에이전트 포털을 통합 구축합니다. 다수 에이전트가 접속하는 환경에서 완전한 데이터 격리, 금융권 수준 보안, 빠른 응답 속도를 보장하는 엔터프라이즈급 시스템 개발이 목적입니다.

2. 사용자 정의

사용자 유형설명주요 기능
관리자(내부)차량·계약·고객·에이전트를 총괄 관리차량 전주기 관리, 계약 상태 전이, 파일 업로드, 에이전트 계정·권한 설정, 현황 대시보드
에이전트(외부 파트너)차량 탐색·견적·계약 요청을 수행신용/저신용 카탈로그, 필터·정렬, 상세 조회, 견적 확인, 계약 요청, 본인 계약 상태 조회

3. 기능 요구사항

3.1 인증 및 보안

  • [필수] 관리자/에이전트 이원화 인증 체계 (JWT)
  • [필수] 에이전트별 테넌트 격리 적용
  • [필수] 민감정보 마스킹 및 권한별 접근 제한(RBAC)

3.2 관리자 백오피스

  • [필수] 차량 전주기 관리, 다중 사진 업로드 및 이력 관리
  • [필수] 계약 상태 전이 처리, 계약서·서류 파일 업로드
  • [필수] 고객 정보·상담 이력 관리, 에이전트 계정·권한 설정
  • [필수] 차량·계약 현황 대시보드

3.3 에이전트 포털

  • [필수] 신용/저신용 탭 분리 카탈로그, 다중 필터·정렬
  • [필수] URL 라우팅 보안 적용 상세 조회, 이미지 리사이즈 스트리밍
  • [필수] 렌탈 조건별 견적 확인, 계약 요청 전송
  • [필수] 본인 계약 목록 조회 및 상태 시각화

3.4 외부 연동 및 성능 최적화

  • [필수] 카카오 알림톡 템플릿 연동·발송
  • [필수] 기존 레거시 시스템 연동 전송 API
  • [필수] 차량 카운트 API Redis 캐싱

5. 기술 제약사항

프론트엔드 React·TypeScript·Vite·Ant Design·Zustand·TanStack Query, 백엔드 NestJS·TypeORM, 데이터베이스 MySQL 8.x·Redis, 인증 JWT를 기준 스택으로 합니다. 모든 코드는 TypeScript strict 모드와 NestJS 모듈 단위 아키텍처를 준수합니다.

6. 범위 외 사항 (Out of Scope)

  • 하자 보수 기간 이후 월 단위 유지보수 (미팅 시 별도 협의)
  • 외부 유료 API의 사용 단가 및 발송 비용
  • 서버 인프라·도메인 비용
  • 본 공고에 명시되지 않은 신규 외부 시스템 연동
